# Gated SRE Agent

An AI SRE agent for small self-hosted teams (k3s/homelab): **Observe** (k8s
events) → **Diagnose** (LLM investigation) → **Gated action** (human-approved
fix). [SPEC.md](SPEC.md) is the source of truth; code that disagrees with it
is wrong until the spec is amended.

The eval harness ships before the agent. It seeds real failures into a k3s
cluster, snapshots the evidence an agent would see, and scores verdicts and
investigation trajectories separately — a right answer reached by luck and a
wrong answer reached carefully are different failures.

## Running the eval harness

Requires a k3s cluster reachable via `KUBECONFIG`, plus `kubectl` on PATH
(teardown uses it). Scenarios run in the `eval` namespace and clean up after
themselves by label selector, even on Ctrl-C.

```sh
python3.12 -m venv .venv && .venv/bin/pip install -r requirements.txt
make eval                                # all scenarios → score baseline → update this README
make eval-one SCENARIO=crashloop_bad_image
make test                                # unit tests (no cluster needed)
```

`make eval` honors the SPEC §3 debounce windows; pass `RUNNER_FLAGS="--debounce-override 0"`
to skip them while iterating.
